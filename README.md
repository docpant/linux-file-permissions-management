# Linux File Permissions Management

## Overview

This project demonstrates Linux file permission management and authorization control using common Linux commands such as `ls -la` and `chmod`.

The project focuses on reviewing existing file permissions, identifying unauthorized access, modifying permissions, securing hidden files, and restricting directory access according to organizational security policies.

## Objective

- Review Linux file and directory permissions
- Interpret Linux permission strings
- Modify permissions using chmod
- Secure hidden files
- Restrict unauthorized directory access
- Apply least privilege principles

## Linux Commands Used

- ls -la
- chmod

## Skills Demonstrated

- Linux security administration
- Access control management
- File permission auditing
- Hidden file management
- Directory authorization
- Least privilege implementation

## Project Structure

- project-description.md
- check-file-permissions.md
- permission-string-analysis.md
- change-file-permissions.md
- hidden-file-permissions.md
- directory-permissions.md
- summary.md

## Outcome

The project successfully identified insecure permissions, updated file authorizations, secured hidden files, and restricted directory access to authorized users only.
